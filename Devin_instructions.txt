Мы разрабатываем проект по моделированию галактик методом Шварцшильда в среде 
https://github.com/GalacticDynamics-Oxford/Agama.git


В этой програмной c++-среде с python-оберткой AGAMA по моделированию галактик методом орбит Шварцшильда на основе forstand-кода Е.Васильева создан код, который ищет наименьшее 
расхождение коэффициентов Гауса-эрмита модельного распределения лучевых скоростей с оценкам этих коэффициентов по наблюдаемым значениям лучевых скоростей 
звезд (параметр penalty в forstand-коде). В коде для фиксированного значения наклонения (incl) производится разбиение картинной плоскости галактики на 
сектора (аппертуры), в которых и производится оценка наблюдаемых и модельных коэффициентов для распределения лучевых скоростей. Далее находятся наилучшие 
значения параметров компонент галактики (axRZ, gammah, rhalo, rho0, дающие наименьше penalty). Для каждого указанного набора параметров кроме того методом 
Брента ищется наилучшее значение Upsilon (масштабный массовый фактор. Для каждой библиотеки орбит можно 
масштабировать грав. поле и соответствующие значения скоростей).
Цель: найти параметры гало ТМ  (axRZ, gammah, rhalo, rho0*Upsilon) и параметры видимой компоненты (incl, Upsilon), дающие наилучшее согласие с наблюдательными
данными (table3.dat и параметры профиля Sersic).
Что уже сделано: проект AGAMA склонирован, код Fornax_P21_symm_PCA_w3Sersic_yaVM.py в папке py запускается на yandex Virtual Mashine 32CPU 32 RAM скриптом 
launch_docker_parallel.sh в той же папке. Для заданного наклонения он ищет наилучшие параметры, дающие наименьший penalty, все это записано и пишется в файлы 
4UpsBoTorch_PCA_Sersic_*.txt и 4result_BoTorch_PCA_Sersic_*.txt. На основе файлов истории мы анализируем границы хороших моделей (80% достоверности и др.
persentiles) в 4х-мерном пространстве параметров для каждого наклонения. Также мы расчитываем J-фактор для каждой модели и строим его взвешенную гистограмму.
